Project Details

We worked with the Dell brand and combined it with the futuristic concept of AR/VR to create a user-friendly, responsive, intuitive website coded with front-end development.

(Disclaimer: This project was a class assignment and is not affiliated with Dell.)

  • My Role: Project Manager, Presentation Coordinator, UX Designer, UX Researcher, Content Writer
  • Tools: Adobe Creative Cloud, InVision Studio, Sketch, Miro, Trello, Google Drive, Excel, Keynote
  • Coding Skills: Visual Studio Code, GitHub, HTML, CSS, JavaScript, JQuery

View Case Study

First Steps:

We compiled data analysis and interviews before forming a framework for early designs.

Proto Personas

It all begins with a story. Empathizing with our users makes them real and relatable.

We set the stage in identifying the problem.

User and Client Interviews

Our team formulated a short but important list of interview questions and surveys, as well as usability tests on the current website.

This step helped us to understand the needs and motivations of both sides.

Defining the User and Problem

We took our research and silently grouped data before analyzing as a group. This gave us a more in depth understanding of users and client.

This enabled us to update the user and client's story to a user persona. From there, we defined the problem and potential solutions.

Next Steps:

We prioritized needs, continued research with competitors, and evaluated the current website.

Feature Prioritization

We had to remain focused on a few key aspects brought out by user and client needs, and condensed our research data even further. This began with the I Like, I Want, I Wish method. Next, we used silent dot voting that turned into a priority matrix to pinpoint the top 3-4 focal points.

Competitor Analysis and BMC

Our next step was taking a detailed look at the strengths, weaknesses, opportunities, and threats to both direct and indirect competitors.

We then developed an overview for our client in a business model canvas.

Information Architecture

Because the client had an existing website, we performed a redline annotation and a heuristic evaluation. Then we finalized new components with card sorting and a sitemap, and started a compilation of inspiration as moodboards.

Onward with User Journey...

As we honed in on a website layout, we returned our focus to the user beginning with a user scenario and storyboard to put ourselves in the user's shoes as they navigate the website, brought out by user testing and research.

From there, we developed a user flow and task flow with 3 main goals to guide our design and set a foundation for the testing to come.

Wireframes and Testing...

Each team member sketched paper wireframes for each user task for both mobile and website. We used A/B testing within the team and with users to determine best features.

With iterations, the paper designs were then turned into clickable low-fidelity, which were then guerilla tested based on a usability and navigation testing plan.

UI, Iterations, and Mobile Responsive Design...

Clickable mid-fidelity prototypes concentrated on UI elements such as color psychology and color testing, information hierarchy, and accessibility. We maintained a style guide for consistency throughout.

Iterations were made after conducting user research and compiling data onto a spreadsheet to analyze the results. This responsive website was also tested as a mobile design.

A high-fidelity prototype was built upon the mid-fidelity, according to testing results, and further testing was done to validate design decisions.

Front-End Development...

We then utilized coding skills to code a final, responsive high-fidelity prototype with another round of testing on both website and mobile.
